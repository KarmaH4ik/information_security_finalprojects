01 — Проблема

Погодные приложения забывают.

Любое потребительское приложение отвечает на вопрос «какая погода прямо сейчас?» и забывает ответ, как только окно закрывается.

  • Нет записи о том, насколько холодно на самом деле было во вторник.
  • Нет способа сравнить города, которые вы проверяли неделями.
  • Нет чистого набора данных, чтобы написать скрипт, построить график или экспортировать.
  • Большинство API закрывают даже базовые запросы за ключами и квотами.

Кому это нужно

Разработчикам, которые живут в терминале; энтузиастам, ведущим журнал локального микроклимата; всем, кто открывал виджет погоды дважды за день и хотел, чтобы он что-то помнил.

Почему сейчас

Open-Meteo предоставляет бесплатные данные о погоде без ключей. Не хватает только маленького, локального и постоянного клиента.

Проблема и мотивация02 / 06
02 — Решение

Одна команда. Два результата.

Один Python-скрипт, который делает ровно две вещи в таком порядке:

  • → stdoutЧистая, читаемая сводка прямо в терминале.
  • → ~/.weather-app-cliЗапись в SQLite, добавляемая поверх предыдущих, с полным сырым ответом API.

Без API-ключей. Одна зависимость (requests). Работает в cron, в скриптах, за shell-алиасами — везде, где работает Python 3.

Один файл

weather.py — семь маленьких функций с единственной ответственностью. Каждый шаг тестируется независимо.

Только добавление

Одна таблица. Ничего не перезаписывается. Сырой JSON хранится целиком — будущие изменения схемы не разрушают данные.

Остальное — ОС

Повторы, расписание, кэш → cron, shell, systemd. CLI намеренно остаётся маленьким.

Архитектура системы04 / 06
04 — Решения

Что выбрали и почему.

ОбластьВыборОбоснование
APIOpen-Meteo (без ключа)Убирает главное препятствие на старте — никаких регистраций, квот и ротации секретов.
ХранениеSQLite, один файлБез настройки, доступно из любого инструмента, тривиально бэкапится. Append-only схема сохраняет историю.
СхемаПлоские поля + raw_jsonГорячие поля индексируются; полный ответ хранится дословно — новые поля можно достать позже без повторного запроса.
ЗависимостиТолько requestsВсё остальное (argparse, sqlite3, json, pathlib) идёт со стандартной библиотекой Python.
Эргономика CLInargs="+" для городаНазвания из нескольких слов (New York) работают без кавычек — соответствует тому, как люди реально печатают.
Ошибкиstderr + exit code 2Чёткое отделение от stdout позволяет инструменту безопасно встраиваться в пайплайны и скрипты.
