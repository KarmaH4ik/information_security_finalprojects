01 — Проблема

Сгенерировать «надёжный» пароль легко сделать неправильно.

Каждый из «очевидных» способов на практике подводит — по-своему.

  • Онлайн-генераторы требуют доверять чужому серверу секрет, который вы только собираетесь использовать.
  • random.choice в Python — Mersenne Twister, детерминирован и предсказуем по выходу.
  • Наивные генераторы часто не проходят политику сайта (нет цифры, нет заглавной) — нужно генерировать заново.
  • openssl rand -base64 даёт байты, но не управляет алфавитом — в выходе живут +/=, ломающие URL и shell.

Кому это нужно

Разработчикам, сисадминам и security-conscious пользователям, которым нужен крошечный, проверяемый скрипт для shell-алиасов, Makefile, playbook'ов Ansible и CI-пайплайнов.

Почему сейчас

Стандартная библиотека Python с 3.6 содержит secrets поверх CSPRNG ОС. Не хватает только маленькой обёртки, которой можно доверять.

02 — Решение

Один файл. Один stdout. Никакой магии.

Один Python-скрипт, который делает ровно три вещи в таком порядке:

  • → argparseРазбор флагов: --length, --lower, --upper, --digits, --symbols.
  • → secrets.choiceКриптостойкий выбор символов из пула включённых классов.
  • → stdoutОдин пароль, одна строка. Ошибки — в stderr, код возврата 2.

Без сети. Без файлов. Без сторонних библиотек. ~100 строк, читается одним проходом.

~/secure-password-generator-cli
$ python3 genpass.py

$ python3 genpass.py --length 24 --no-symbols

# один файл, ноль зависимостей,
# случайность из CSPRNG ОС.

03 — Архитектура

Линейный конвейер из четырёх стадий.

Входargv (флаги CLI)
→--length, --lower, --upper…
Шаг 1parse_args()
argparse + BooleanOptionalAction
Шаг 2build_pool()
выбор классов символов
↓pool + список классов
Выходпароль в stdout
Финалmain() — print + exit code
←N символов + перемешивание
Шаг 3generate_password()
secrets.choice + SystemRandom

Один файл

genpass.py — четыре функции с единственной ответственностью. Каждый шаг тестируется независимо.

Ноль зависимостей

Только стандартная библиотека: argparse, secrets, string, sys. Никакого requirements.txt.

Гарантия + перемешивание

По одному символу каждого класса в первые слоты, затем SystemRandom().shuffle — политика без перегенерации.

04 — Решения

Что выбрали и почему.

ОбластьВыборОбоснование
Случайностьsecrets, не randomCSPRNG ОС (/dev/urandom, BCryptGenRandom); устойчив к восстановлению состояния по выходу.
ЗависимостиТолько stdlibОдин файл, тривиальный аудит, ноль supply-chain рисков. Можно скопировать в любой репозиторий и запустить.
Алфавит символовСвой SYMBOLSИсключены ', ", `, \ — символы, ломающие shell, SQL и копи-пейст.
ПолитикаГарантия + перемешиваниеПо одному символу каждого класса + SystemRandom().shuffle: предсказуемое время, без перегенераций.
Эргономика CLIBooleanOptionalActionПарные флаги --symbols / --no-symbols без ручного дублирования action-классов.
Ошибкиstderr + exit code 2Конвенция CLI: stdout остаётся чистым для пайпов, ошибки использования — отдельным каналом.
